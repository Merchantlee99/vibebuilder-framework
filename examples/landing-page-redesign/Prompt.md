# Prompt.md

## Working Title

- Meeting Briefing Landing Page Redesign

## Goal

- 기존 랜딩페이지를 다시 설계해 제품 가치가 더 빨리 이해되고, 데모 요청 CTA가 더 분명하게 보이도록 만든다.

## Non-Goals

- 새 가격 정책 설계
- 백엔드 기능 추가
- CMS 도입
- 다국어 지원

## User Problem

- 현재 랜딩페이지는 시각적으로 무난하지만, "무엇을 해결하는 제품인지"가 5초 안에 전달되지 않는다.
- 핵심 사용자 pain보다 기능 목록이 먼저 보여서 방문자가 데모 요청까지 가지 않는다.

## Constraints

- 이번 라운드는 프론트엔드 리디자인만 다룬다.
- 기존 코드베이스와 스타일 시스템을 최대한 재사용한다.
- 구현은 1 sprint 안에서 끝낼 수 있어야 한다.

## Inputs We Already Have

- 기존 랜딩페이지 초안
- 제품 핵심 메시지 3개
- 고객 인터뷰 메모에서 뽑은 pain point

## Unknowns

- 어떤 CTA 문구가 가장 전환에 유리한지
- testimonial 섹션을 이번 라운드에 넣을지

## Deliverables

- 새 히어로 섹션 구조
- pain-to-solution 흐름이 보이는 본문
- 명확한 CTA 배치
- 모바일에서도 깨지지 않는 레이아웃

## Done-When

- 첫 화면만 봐도 타깃 사용자, 문제, 제품 가치가 이해된다.
- CTA가 한 화면 안에서 분명히 보인다.
- 모바일과 데스크톱에서 주요 레이아웃 문제가 없다.

## Notes

- 이번 라운드는 "더 화려하게"가 아니라 "더 분명하게"가 기준이다.
